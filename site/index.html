Local-first credential launcher

Replace repository secrets with runtime-only credentials.

EnvVault keeps long-lived provider keys in the OS credential store, then resolves repository-safe envvault:// references only when launching a child process.

Concepts

Credential Flows

Pick the narrowest flow your tool supports. Proxy mode keeps the real provider key out of the child process; raw injection is the fallback for SDKs that cannot use a localhost endpoint.

Provider proxy

Use this for OpenAI-compatible APIs and SDKs that accept a custom base URL. 

ENVVAULT_PROXY_URL=envvault://gemini-openai/dev/base-url
ENVVAULT_PROXY_TOKEN=envvault://gemini-openai/dev/token

/base-url resolves to a dynamic localhost proxy URL for the selected profile. It is documented as a proxy URL because the actual value changes each run.

Raw inject

Use this when an SDK requires the real provider key in code or in an environment variable. 

GEMINI_API_KEY=envvault://gemini/dev/value
GEMINI_MODEL=gemini-3.5-flash

/value intentionally places the raw credential in the child process environment.

First-party process token

Use this for local backends that verify EnvVault-issued JWTs. 

BACKEND_A_TOKEN=envvault://backend-a/dev

Scope, resource, TTL, and project binding come from local trusted profile policy, not from repository files.

Run it locally

Quickstart

1

Build the CLI

go build -o ./bin/envvault ./cmd/envvault
2

Store a provider credential

printf 'YOUR_GEMINI_API_KEY\n' | ./bin/envvault credential add gemini-api-key \
  --value-stdin
3

Create a proxy profile

./bin/envvault proxy add gemini-openai/dev \
  --credential gemini-api-key \
  --provider openai-compatible \
  --target https://generativelanguage.googleapis.com/v1beta/openai \
  --allow-path /chat/completions \
  --allow-method POST \
  --project-binding none
4

Launch through EnvVault

./bin/envvault exec --env-file examples/gemini-ai-sdk-proxy-app/.env -- \
  npm --prefix examples/gemini-ai-sdk-proxy-app start

Security model

What EnvVault protects

EnvVault reduces repository and child-process exposure. It does not sandbox the launched process, redact application logs, or protect a compromised OS credential store.

  • Repository files contain references, not long-lived values.
  • Proxy profiles attach real provider keys only after allowlist checks.
  • Raw inject is explicit and should be used only when proxying is not possible.
  • The admin UI lists metadata but does not display stored credential values.
